Memo Note Live Demo

close
A spatial workspace for digital brainstorming.

Role: UX/UI Designer & Front-end Developer

Stack: React, Redux, Firebase, Figma, Material-UI, etc.

The Challenge

Most note-taking applications follow a rigid, linear list format. While organized, this format doesn't mimic the way the human brain brainstorms, which is often non-linear and spatial. Users often feel "boxed in" by standard interfaces, losing the creative freedom of physical sticky notes on a whiteboard.

How might we create a digital note-taking experience that combines the flexibility of physical movement with the power of cloud synchronization?

The Objectives

  • Spatial Freedom: Empower users to prioritize information through size and position.
  • Real-time Continuity: Ensure that a thought captured on one device is instantly available and formatted on another.
  • Visual Efficiency: Use color-coding and filtering to reduce cognitive load when managing large volumes of notes.

Building User Empathy

Persona Development

  • Persona 1: The Efficient Project Lead

    Name: Sarah

    Age: 33

    Role: Project Manager at a Tech Startup

    Tech Literacy: High

    Goal: To turn a messy morning stand-up meeting into a structured, prioritized task board.

    The Scenario: Sarah starts her day with 15 different tasks coming from email, Slack, and meetings. She needs a "scratchpad" that feels more flexible than a rigid Jira board but more permanent than a physical post-it.

    How she uses Memo Note:

    • Spatial Hierarchy: She resizes "High Priority" memos to be twice the size of "Backlog" memos so they catch her eye immediately.
    • Color Logic: She uses the Color Filter religiouslyâRed for "Blockers," Green for "Completed," and Blue for "Reference."
    • The "Clean Up": At the end of the day, she uses the Archive feature to clear her workspace without permanently deleting her history.
  • Persona 2: The Visual Creative / Student

    Name: Leo

    Age: 21

    Role: Design Student & Freelancer

    Tech Literacy: Moderate

    Goal: To mood-board ideas and keep track of inspiration "snippets" in a non-linear way.

    The Scenario: Leo is constantly finding quotes, color hex codes, and quick design reminders. He avoids folders, preferring to see everything at once to find connections between his ideas.

    How he uses Memo Note:

    • Free-form Layout: He uses the Drag-and-Drop feature to group related ideas together in clusters on the screen.
    • Mobile-to-Desktop Sync: He often captures a quick thought on his phone using the Responsive Web Design layout while on the bus, then re-organizes those notes on his 27-inch monitor when he gets home.
    • Visual Discovery: He uses the Text Filter to quickly find a specific hex code he saved weeks ago among a "sea" of 50+ notes.

User Journey Map

This journey maps the psychological transition from Information Overload to Mental Clarity using the spatial and organizational tools of Memo Note.

Functionalities

  • Create a new memo.
  • Update memo titles, content, and background color.
  • Resize memos to adjust the width and height.
  • Drag and drop memos within the canvas.
  • Archive memos.
  • Delete memos.
  • Store memo layout in localStorage.
  • Sync memo updates and archives with Firebase.
  • Show a Snackbar (Toast) to notify users when memo actions are completed.
  • Filter memos by color or text.

Design & Prototype

Visual Identity & Material Design

I chose Material-UI (MUI) as the core design system. This choice wasn't just aesthetic; it was functional.

  • Elevations: I used MUI's "Paper" elevations (shadows) to make notes look like they are floating above the canvas, subtly suggesting they can be picked up and moved.
  • Color Palette: I created a palette of 6 distinct memo background colors. These pastel colors were tested for text readability to ensure that even with dark text on a colored background, the notes remained accessible.
  • Typography: Using Open Sans provided a clean, modern look that remains legible even when notes are resized to their smallest dimensions.

Wireframing with Figma

To bridge the gap between conceptual ideas and visual structure, I crafted low-fidelity wireframes with Figma. This skeletal framework allowed me to iterate quickly on functional layouts and content hierarchy. This stage also focused on defining the spatial relationships between UI elements and mapping out the essential pathways for our primary personas.

Prototypes & Interaction

The "Magic" of Memo Note happens in the interaction. I focused on the physics of the UI.

  • Drag & Resize (The "Snap" Effect): Using react-grid-layout, I implemented a grid system where notes snap into place. This provides a satisfying "tactile" feel. When users drag a note, the other notes fluidly reflow.
  • Feedback Loops: Every action (creating, updating, archiving, or deleting) is followed by a MUI Snackbar (Toast). This is crucial for Sarah, who needs confirmation that her work is saved to the cloud, and for Leo, who needs to know his "Delete" was successful.
  • Quick filter: To speed up filtering, we use color buttons instead of dropdowns to save an extra click. And the input field updates instantly via "onchange" without a submit button.

Project Reflection

What's Next?

  • Markdown Support: Transitioning from plain text to Markdown Editor.
  • Image Attachments: Allowing users to drop images onto the grid alongside their text notes.
  • Dark Mode: Implementing a dark theme for late-night to reduce eye strain.
  • Folders & Tags: Allowing users to further organize notes with folders and tags.
  • Data Recovery: Building a Trash page for deleted memos and adding an Undo function to prevent accidents.
  • Keyboard Shortcuts: Implementing a global "Enter" shortcut to create notes even faster.

Project Takeaways

  • UX/UI is a Balancing Act: I learned that while "unlimited freedom" sounds good, a grid system actually helps users feel more organized.
  • User-Centric Feedback: The addition of Snackbar (Toast) changed the feel of the app from "static" to "responsive and alive."
  • State Management is Key: Using Redux was essential for keeping the memo data in sync with the Firebase backend without performance lag.
  • Performance in the DOM: Managing a large number of draggable elements taught me how to optimize React re-renders and use Redux effectively to prevent lag.

Movie World Live Demo

close
A streamlined discovery experience for film enthusiasts.

Role: UX/UI Designer & Front-end Developer

Stack: Cypress, React, Redux, React Router, Axios, React Reveal, etc.

The Problem

Existing movie database platforms (IMDb, Rotten Tomatoes) are often criticized for cluttered layout with excessive advertisements and complex navigation that overwhelms users. These heavy "noise", including performance latency, create a frustrating UX for film enthusiasts seeking a clean, fast-moving interface.

How might we create a lightweight discovery experience with clean UI?

The Goal

To bridge the gap between cinematic immersion and high-performance engineering by delivering a distraction-free, media-first environment for film discovery.

  • ⚡ Lightweight Speed: Eliminating the "noise" of traditional databases to ensure a frictionless user journey where content is always two clicks away.
  • 🎬 Visual Storytelling: Prioritizing high-quality posters and trailers as the primary UI language to create an immersive, cinematic atmosphere.
  • 🏗️ Enterprise Integrity: Building with a modern React architecture to ensure the code is as scalable and performant as the visual experience.

UX Research & Strategy

User Insights

Through informal user interviews with movie buffs, I identify three primary "pain points":

  • 😵‍💫 Information Overload: Excessive text and cluttered metadata create "visual noise" that distracts from discovery. Enthusiasts find it easier to connect with a movie through its imagery and trailers rather than dense blocks of statistics.
  • 📱 Mobile Friction: Many sites fail to provide a fluid browsing experience on smaller viewports. Users need a touch-optimized, fluid experience that prioritizes screen real estate.
  • 🎯 Fragmented Discovery: Users expressed frustration with having to leave a movie site to search for trailers or reviews elsewhere. What if I centralize multimediaâintegrating YouTube trailers and community reviews directly into the movie's detail page to eliminate "context switching"?

Information Architecture

Use a "shallow" structure to ensure movie info is less than two clicks away. This efficient UX reduce the cognitive load from users. It eliminates the friction that often kills engagement in content-heavy apps.

Design Process (UI)

Design Philosophy

Efficiency Through Minimalism
My approach for this project was to eliminate "interaction friction". I focused on creating a seamless interface where the technology anticipates the user's intent. The goal was to transform the traditional "Search, Click, Wait" pattern into a fluid and reactive discovery experience.

User Flow Summary

The core of this UI is a reactive search-and-discovery loop designed to get users to their content with the fewest possible clicks.
  • 🚀 Immediate Engagement: Upon landing, the interface removes the need for a "starting click" by auto-focusing the input field, allowing users to begin their journey instantly.
  • 🔍 Predictive Results: By replacing the traditional submit button with an onChange mechanism, this ensures the UI syncs with the user's input instantly, giving the data retrieval a seamless and "live" feeling.
  • 📐 Adaptive Visuals: A dynamic grid system (2-5 columns) ensures the movie posters remain the hero of the experience across all devices, maintaining visual density without clutter.
  • 🍿 Contextual Depth: A deep-dive experience without the clutter. This layout aggregates critical movie data (ratings, trailers, and reviews) into a single responsive container. This UI eliminates unnecessary navigation while upholding minimalism.

Design System

  • 🎨 Color Palette: I choose a dark theme (Deep blue-gray background) to make the movie posters pop, using lightgray and champagne accent to offer a warm and inviting look.
  • 🔠 Typography: Utilizing Open Sans, a clean and sans-serif font, to ensure excellent readability on various devices. This font creates a welcoming and approachable experience.
  • 🌊 Transitions: I integrate React Reveal to create a "fade-in" effect as users scroll. It signals the "start of a scene" for that specific film, easing the user into the content rather than jarring them with a sudden visual "cut".

Iterations

Early prototypes utilized a single-page "pop-up" for movie details, but user testing revealed a "claustrophobic" browsing experience. I pivoted to a React Router architecture to implement a multi-route system, giving every film its own dedicated "stage". This transition not only improved visual focus but also enabled deep-linking, allowing users to share specific cinematic discoveries directly.

Engineering Challenges

  1. SPA Routing Persistence

    The Problem
    Hosting on GitHub Pages presented a challenge for deep-linking. Using a standard BrowserRouter would cause the server to return a 404 error upon page refresh, as the static server looks for a physical file that doesn't exist for unique movie URLs.
    The Solution
    I implemented HashRouter to manage the application's navigation. By utilizing the hash portion of the URL (e.g., /#/movie/11), the browser ignores the path during server requests and allows React Router to intercept the change via JavaScript.
    The Result
    This implementation ensures users refresh any movie detail page without losing their session. By leveraging client-side rendering, I eliminated "white screen" flickers to create a fluid, uninterrupted experience while maintaining a stable, complex routing structure within a static hosting environment.

  2. Optimizing API Efficiency

    The Problem
    By default, the onChange mechanism triggered an API request for every single keystroke. This resulted in excessive network traffic and "UI flickering," where the loading spinner and search results would rapidly flash as the user typed, forming a jittery and distracting experience.
    The Solution
    I implemented a Debouncing strategy to delay the API call until the user has finished typing for 500ms. This ensures that data is fetched once the user's intent is clear, significantly reducing redundant network overhead.
    The Result
    The implementation created a much smoother UI by eliminating the aggressive flickering of the loading state. By streamlining the data-fetching logic, I reduced API load and provided a more polished, professional feel to the search discovery process.

  3. Improving URL Semantic Clarity

    The Problem
    Originally, movie pages were identified solely by a numeric ID (e.g. /#/movie/424). This made shared links look suspicious or uninformative to users and provided zero keyword value for search engine indexing.
    The Solution
    I engineered a createSlug utility function to transform movie titles into URL-friendly strings. The logic cleans the data by converting "&" to "and," stripping special characters, and collapsing spaces into single hyphens. This "slug" is then appended to the movie ID, creating a descriptive, human-readable URL.
    The Result
    The final URLs (e.g. /#/movie/424/schindlers-list) provide immediate context to the user before they even click. This improves social shareability, enhances SEO keyword relevance, and ensures a more professional, "content-first" navigation structure.
